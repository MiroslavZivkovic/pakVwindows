WinMUMPS

WinMumps is a set of project files allowing to build MUMPS on Windows 32- and 64-bits.
These projects are developed and currently used by Free Field Technologies. So, fell free to modify them if necessary. For more informations, you can send a mail to mathieu.gontier@fft.be

Requirements

MUMPS

    MUMPS-4.6.4 or latter.

Compilation

    C/C++ build environment:

    • either Visual Studio 2003 Professional;
    • or Visual Studio 2005 Professional (not patched with Visual Studio 2005 Service Pack 1).

    Fortran build environment:

    • Intel Fortran compiler 9.x.

Parallelism

    For the parallel version, the compilation is based on the following MPI libraries:

    • MPICH-1.2.5 for Windows 32-bits;
    • MPICH2-1.0.3 for Windows 64-bits.
    These libraries have to be installed with the default parameters (for instance the installation path). If this requirement is not followed, the project files should be modified.

    Moreover, MUMPS parallel requires BLACS and SCALAPACK distribution. WinMumps provides project files to build them. Copy these distributions in the same directory than MUMPS if you do not want to modify the project files by yourself.

Installation

    Just copy the WinMumps directory in the main directory of MUMPS.

    One should organize MUMPS and its dependencies as following:

      directory-on-the-host-machine
        |_ MUMPS-version
          |_WinMumps
          |_ ...
        |_ blacs
        |_ scalapack
    It will permit to work correctly without any modification of the project files.

Python scripts

    Some Python scripts are distributed with WinMumps. They are mainly used for an easier deployement of the provided project files.
    Note that Python-2.5-release is required.

convert_msnet2005to2003.py

    WinMumps is mainly developed with Visual Studio 2005 professional on Windows-x86-64. The script convert_msnet2005to2003.py allows to convert Visual Studio 2005 projects supporting both 32- and 64-bits compilations to Visual Studio 2003 projects supporting 32-bits compilation only.
